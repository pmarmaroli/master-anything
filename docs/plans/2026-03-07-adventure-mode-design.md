# Adventure Mode Design

## Problem

The app's educational chat format doesn't engage teenage users. A 16-year-old fan of Nintendo games (Zelda, Pikmin, Mario Kart) loses interest at two points: no hook at the start, and the Q&A format becomes monotonous after a few exchanges.

## Solution

A toggleable **Adventure Mode** that transforms the learning experience into a narrative quest with collectible rewards, while keeping the exact same educational engine underneath. A **Study Mode** toggle preserves the current serious experience.

## Design

### Toggle (Study / Adventure)

- Button in the header next to the existing Reading/Listening toggle
- Two states: Study (book icon) / Adventure (sword icon)
- Stored in session state, sent to backend with each request
- Study mode = current behavior, zero changes
- Persistent within session

### Agent Personas (Adventure Mode)

When adventure mode is active, each agent adopts a character via additional prompt instructions:

| Agent | Persona | Emoji | Narrative Role |
|-------|---------|-------|----------------|
| Architect | Le Cartographe | Map | Traces the quest map, reveals zones to explore |
| Mentor | Le Sage | Wizard | Wise guide who teaches secrets |
| Challenger | Le Gardien | Sword | Blocks passage, poses riddles to advance |
| Naive Student | Le Compagnon | Paw prints | Curious little creature asking for simple explanations |
| Evaluator | L'Oracle | Crystal ball | Judges if the hero is ready for the next stage |
| Orchestrator | Le Narrateur | Scroll | Handles narrative transitions |
| Renderer | L'Artisan | Palette | Creates quest maps and illustrations |

The AI improvises the narrative tied to the learning subject (e.g., electricity = "Cave of Electrons"). Tone: fun, encouraging, slightly epic, never patronizing. Educational rigor is unchanged.

### Collection System (Inventory)

Each mastered concept (evaluator decision = "advance") grants a collectible:

- **Generated by the Oracle**: name, emoji, 1-line description tied to the subject
- Added to evaluator JSON response as a `"reward"` field:
  ```json
  "reward": { "name": "Sparkling Electron", "emoji": "lightning", "description": "Captured in the Cave of Electrons" }
  ```
- Stored in `session.inventory` (backend)

### Inventory UI

- Button in header (backpack icon) — visible only in Adventure mode
- Modal with emoji grid + names (minimalist Pokedex style)
- Each item shows its associated concept on hover/tap

### Celebration

- Existing `MasteryBadge` component enhanced: displays "You found the [emoji] [name]!" instead of generic mastery message
- Small animation on collection

### Prompt Engineering

No new agents or complex code. A conditional block is appended to each agent's prompt in `getSystemPrompt()` when `adventureMode === true`:

- Persona description and narrative tone
- Instruction to tie the narrative to the learning subject
- Instruction to keep educational content rigorous — only the tone changes
- For the Evaluator: instruction to include `reward` field in JSON on advance

### Backend Changes

- Add `adventureMode: boolean` to `SessionState`
- Add `inventory: Reward[]` to `SessionState`
- `Reward` type: `{ name: string; emoji: string; description: string; concept: string }`
- Pass `adventureMode` in API request body
- Conditional prompt enrichment in `OrchestratorService.getSystemPrompt()`

### Frontend Changes

- Adventure/Study toggle in header
- `adventureMode` state in `useChat`, sent with API requests
- `InventoryModal` component (modal with collected items grid)
- Backpack button in header (adventure mode only)
- `MasteryBadge` enhanced with reward display

### Backward Compatibility

Study mode = 100% current behavior. Adventure mode is purely additive.
